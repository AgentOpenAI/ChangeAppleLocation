### 2. WLOC 报文深度字段映射与属性定义说明

#### A. WLOC 包外层信封（Envelope Header）
在客户端与服务器的传输协议中，前 10 个字节通常是苹果私有的传输通道控制头，用于描述语言代码、客户端 App Bundle 名（如 `com.apple.locationd`）以及随后的 Protobuf 数据包的字节长度（大端存储，如 `01 3b` 代表 315 字节）。

#### B. 核心业务字段映射表格

| Protobuf 字段编号 (Field No) | 数据类型 (Wire Type) | 推荐对应属性名 | 字段功能描述 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **`Envelope Field 2`** | `2` (Length-delimited) | `wifiDevices` | 周围 Wi-Fi 路由器的定位热点信息列表 | 客户端可包含数百条此消息 |
| **`Envelope Field 22 / 24`** | `2` (Length-delimited) | `cellTowers` | 蜂窝基站定位列表信息 | LTE 基站信息、4G/5G 等 |
| **`WiFi Struct Field 1`** | `2` (Length-delimited) | `bssid` | Wi-Fi 设备的 MAC 硬件地址 | 标准 MAC 格式：`xx:xx:xx:xx:xx:xx` |
| **`WiFi Struct Field 2`** | `2` (Length-delimited) | `location` | 该 MAC 地址对应苹果服务器算出的物理坐标子消息 | **重点篡改/扫描的对象** |
| **`Cell Struct Field 5`** | `2` (Length-delimited) | `location` | 该基站对应苹果服务器算出的坐标子消息 | **重点篡改/扫描的对象** |
| **`Cell Struct Field 1`** | `0` (Varint) | `mcc` | 移动国家代码（Mobile Country Code） | 中国一般为 `460` |
| **`Cell Struct Field 2`** | `0` (Varint) | `mnc` | 移动网络代码（Mobile Network Code） | 移动 `00`/`02`，联通 `01`，电信 `03`/`11` |
| **`Cell Struct Field 3`** | `0` (Varint) | `lac/tac` | 位置区码（Location Area Code / Tracking Area Code） | 描述基站逻辑区域 |
| **`Cell Struct Field 4`** | `0` (Varint) | `cellId` | 基站小区编号（Cell ID） | 用于准确定位某个具体基站发射塔 |
| **`Location Struct Field 1`** | `0` (Varint) | `latitude` | **纬度**（乘以 $10^8$ 后的小数） | 例如 3123041600 代表纬度 `31.230416` |
| **`Location Struct Field 2`** | `0` (Varint) | `longitude` | **经度**（乘以 $10^8$ 后的小数） | 例如 12147370100 代表经度 `121.473701` |
| **`Location Struct Field 3`** | `0` (Varint) | `accuracy` | **水平定位精度**（单位：米） | 路由器精度多在 15-50 米，基站多在 300-1000 米 |
| **`Location Struct Field 4`** | `0` (Varint) | `altitude` | 海拔高度/垂直精度信息（若支持） | 常常为空或不提供 |

## 二、iPhone 上的多套定位源与 App 最终拿到的数据

### 1. iPhone 上的“多套”定位源
对操作系统而言，iOS 的底层服务的确存在四种截然不同的原生定位源：
1. **GPS/GNSS 卫星硬件定位**：GPS、北斗、GLONASS 等卫星定位。无需联网，由物理定位芯片直接与卫星建立通信。
2. **基站定位（Cell ID）**：通过手机连接的移动网络基站的信号强度 and 位置来确定坐标。
3. **Wi-Fi 定位（WLOC）**：扫描周围 Wi-Fi 路由器的 MAC 地址列表并向苹果 WLOC 服务器（`gs-loc.apple.com`）检索坐标。
4. **蓝牙及 IP 辅助定位（Beacon / GeoIP）**：极其微弱或起辅助参考。

### 2. 系统中枢：融合定位系统（Fused Location Provider）
**虽然底层有四套定位来源，但 App 无法在系统里分别、直接去读取其中的某一套原生原始数据。**

所有 App（微信、高德、滴滴、美团等）要获取定位，都必须通过统一的高层框架 `CoreLocation` 申请权限：
* 系统级的守护进程 `locationd` 才是直接管控 GPS 芯片和网络请求的“大总管”。
* `locationd` 会启动**融合定位算法**：它是一个高精度的位置过滤器（如卡尔曼滤波）。它根据当时芯片的耗电情况、各定位源的当前精度反馈（Accuracy）、运动传感器状态，自动融合出一份**最高精度的、唯一的、经过系统校验处理的坐标对象（`CLLocation`）**，然后分发给 App。

### 3. App 最终拿到的数据内容 (CLLocation)
App 拿到的并不是“我的 GPS 坐标是 XXX，我的网络坐标是 YYY”，而是由系统合成好的 `CLLocation` 对象：
* **`coordinate`**：经纬度（`CLLocationCoordinate2D`，包含由系统转换好的经度、纬度）。
* **`horizontalAccuracy`**：水平精度半径（米）。如果系统使用了高精度的 GPS 定位，这个值会很小（如 3 米 - 10 米）；如果系统迫于信号不佳，退化采信了网络定位，这个值会变大（如 25 米 - 65 米）。
* **`altitude` & `verticalAccuracy`**：海拔高度和垂直精度（GPS 信号好时才有准确数值）。
* **`timestamp`**：本次定位生成的时间戳。
* **`speed` & `course`**：当前移动的速度（米/秒）以及运动航向的角度（0°~359.9°）。

## 三、App 能否自行选择定位源或直接访问 GPS 底层数据？

### 1. App 能否控制使用网络定位还是 GPS 定位？
**不能直接控制。**
App 无法在 API 里指定“我只要 GPS”或“我只要 WiFi 辅助定位”。iOS 的 CoreLocation 框架仅向第三方 App 提供了一种“间接控制”——通过 `desiredAccuracy`（期望精度）和 `distanceFilter`（距离过滤器）属性向系统提出申请。

| App 设置的 `desiredAccuracy` 参数 | 系统底层可能触发的定位模式 |
| :--- | :--- |
| `kCLLocationAccuracyBestForNavigation` (车载导航) | **系统必定强制唤醒 GPS 芯片进行不间断全功率工作。** 此时高德、腾讯地图在室外能拿到最实时的 10 米以内 GPS 轨迹。 |
| `kCLLocationAccuracyBest` (日常精确定位) | 系统在户外尽量用 GPS，在室内无法连接卫星时无缝回退到网络 WLOC（WiFi/基站）定位。 |
| `kCLLocationAccuracyHundredMeters` (百米精度) | 系统为省电根本不会开启高能耗的 GPS 芯片，而是直接读取 WLOC 或蜂窝基站数据快速算一个低精度坐标。 |
| `kCLLocationAccuracyThreeKilometers` (三公里精度) | 几乎完全依靠运营商基站或 IP 粗略定位，能耗极低。 |

### 2. 微信、高德地图等 App 内部是如何处理的？
微信、高德等 App 在内部会有自己的**二次过滤和容错算法**（App-level Fused Logic）：
* 高德地图在导航模式下，不仅读取 CoreLocation 的坐标，还会读取陀螺仪、加速度计和指南针（惯性导航），并结合高德自家的路网（Map Matching）进行道路贴合纠偏。
* 微信在进行摇一摇、附近的人、朋友圈定位时，会判断 `CLLocation` 携带的 `horizontalAccuracy`（精度半径）：
  - 如果发现精度高达 $25$ 米以内（如 GPS），则认为是可信度极高的物理运动位置。
  - 如果精度在 $3000$ 米甚至更大，微信会判定当前位置精度过低，提示用户“请开启 GPS 提升精度”或在界面上列出更大的区域。

## 四、如何保证设备 100% 走网络定位（避免走真实 GPS 定位）？

如果您正在使用该 WLOC 篡改模块（通过代理修改网络定位），想要让修改**完全生效、绝不回退到真实 GPS 导致定位飘回真实地**，关键就在于**断绝系统对 GPS 卫星物理信号的接收**。

以下是实现“100% 仅采信网络定位”的实战方案：

1. **绝对物理屏蔽（最稳妥）**：
   * 在**建筑内部、高大地下室、金属防弹屋**内进行测试。这些地方有厚实的钢筋水泥结构和金属阻挡，GPS 信号完全无法穿透，系统在 0 颗卫星连接的情况下，100% 强制信赖网络/WiFi定位（即我们拦截篡改好的 WLOC 信息）。
2. **关闭 GPS 物理模块（通过飞行模式）**：
   * 开启手机的“飞行模式”（关闭基站信号，防止 Cell 辅助定位定位到真实城市基站）。
   * 仅连接室内的 Wi-Fi 路由器，并连上您的 Surge 等代理代理工具。
   * 此时手机没有运营商蜂窝，在室内也无卫星，系统的定位器 100% 仅信赖正在走代理的 WLOC WiFi 坐标。
3. **针对 iOS 26/27 高版本定位缓存的特别操作（极重要）**：
   * iOS 会对上一次搜到的真实 GPS 坐标在 `locationd` 进程中做长达十几分钟的内存强缓存，即使到了室内或者开了飞行模式，可能依然读取缓存。
   * **完美解决步骤**：
     1. 首先在 Worker 选点网页上选好目标坐标并“储存到设备”；
     2. 开启飞行模式 → 关闭系统的“定位服务”开关；
     3. **重启手机**（强制将 `locationd` 守护进程的内存缓存彻底擦除）；
     4. 重启后开机，连接 WiFi → 确认代理工具已启动且 VPN 标志显现；
     5. 开启“定位服务”开关，打开高德/微信，定位 100% 瞬移到篡改的网络位置，极其稳定。

## 五、在普通手机（无越狱、无开发者模式）上，如何拦截修改 GPS 定位？

有些时候我们必须在室外移动，或者 App 正在强制请求 GPS 底层高精信号（例如高德、打卡App）。此时通过代理修改网络 WLOC 就无能为力了。
针对**普通无越狱、无开发者的手机**，目前工业界和个人爱好者最推荐的 2 种能够物理劫持修改真实 GPS 的通用方案：

### 方案 1：外接物理 MFi 硬件定位外设（双开定位器）—— 推荐，普通用户首选
* **原理**：
  这种外设利用了苹果 MFi（Made for iPhone）官方外设认证协议。苹果允许经过授权的外部导航硬件（例如外接的高精度车载 GPS 模块、测绘级天线）通过闪电（Lightning）或 Type-C 接口连接 iPhone，并通过 CoreLocation 框架将外接 GPS 的坐标直接写入系统底层，覆盖 iPhone 自带的板载 GPS 芯片。
* **硬件外观**：
  市面上常见的“定位神器”（如一个小巧的尾插，插在充电口上，配合专门的控制 App）。
* **免开发者、免越狱操作流程**：
  1. 将硬件插入 iPhone 充电接口。
  2. 配合的控制 App（通常通过证书签名安装，不需开发者模式）在地图上任意选点。
  3. 硬件芯片开始模拟物理 GPS 信号，向 iOS 系统的 `locationd` 广播虚拟 GPS 坐标。
  4. 系统认为高精度的外接 GPS 才是最可信的硬件，从而让系统中的所有 App（包括高德、微信、钉钉等）100% 拿到的都是修改后的硬件坐标。
* **优缺点**：
  * **优点**：不需要电脑，即插即用，稳定性极高，100% 覆盖 GPS 硬件级，任何户外和室内场景均完美支持。
  * **缺点**：需要花钱购买物理硬件硬件（价格在百元左右），设备体积略微增大。

### 方案 2：利用电脑进行电脑端“一键修改位置”（AnyGo/爱思助手/iMyFone等）
* **原理**：
  苹果官方的开发套件中，允许通过电脑端调试协议（DeviceLink 接口）向连接 of iOS 设备发送虚拟位置（GPX 路径文件），供开发者调试 App 地图功能。各大助手和定位工具就是利用了这个官方接口，将电脑上设定的虚拟坐标注入手机底层的 `locationd`。
* **免越狱操作流程**：
  1. 手机不需要越狱，但根据 iOS 16 及以上系统版本的要求，手机端需要进入 `设置` -> `隐私与安全性` -> **开启“开发者模式”**（不需要个人注册开发者账号，人人皆可在设置里一键开启并重启手机即可）。
  2. 使用数据线将 iPhone 连接到电脑。
  3. 启动电脑端的定位工具（如爱思助手 -> 工具箱 -> 虚拟定位；或者专业的 AnyGo、iMyFoneAnyTo）。
  4. 在电脑端地图上搜索上海外滩或点击修改。
  5. 此时电脑软件通过 USB 数据通道下发坐标指令，手机底层 GPS 被全局接管、永久定格在该坐标（即使拔掉数据线也依然生效）。
  6. **恢复真实定位**：在电脑端软件点击“还原”，或者直接将手机重启一次，便会瞬间恢复真实的物理定位。
* **优缺点**：
  * **优点**：完全免费，软件极易操作，可模拟运动轨迹、循环路线，不需要购买任何外设。
  * **缺点**：iOS 16+ 虽然人人都可自行在手机设置里点开“开发者模式”开关（不需要花钱，只点个开关重启），但必须有电脑配合；且手机只要重启，定位就会恢复真实，无法实现脱离电脑在外面随时随地切换定位。
